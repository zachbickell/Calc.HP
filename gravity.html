Info

The 3 input groups below are for each of the respective equations above. Any values entered in scientific notation must be entered using the e operator instead of multiplying by 10.
Example: 2.5 ⋅ 105 must be entered as 2.5e5. The same applies for negative exponents. 3.2 ⋅ 10-11 must be entered as 3.2e-11.
This tool is kept in fixed metric units to avoid any inaccuracies due to rounding that occurs with conversion. Results in scientific notation are not limited to any amount of significant figures below the JS limit. You can toggle rounding for all 3 functions, though it is recommended you keep the Force function unticked. The rounding parser in JS will automatically round small numbers represented by e-(integer) to 0, which is inaccurate.
